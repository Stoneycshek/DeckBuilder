# 遊戲設計總結文件（Design Draft）

## 1. 專案目標與核心理念

本遊戲為 **Roguelike × Deckbuilder** 類型，核心特色為：

> **卡牌同時具備「可直接使用的技能」與「可拆解、可重組的零件」雙重身份。**

玩家在戰鬥中使用完整技能卡牌，在戰鬥外（合成商店）則需做出策略性抉擇：
- 保留卡牌作為即戰力
- 或拆解、合成以換取長期強化

設計目標：
- 高重玩性（High Replayability）
- 高策略密度（Meaningful Decisions）
- 少量基礎元素 × 高組合深度

---

## 2. 遊戲核心循環（Core Loop）

1. 進入戰鬥
2. 使用卡牌進行回合制戰鬥
3. 獲得卡牌 / 資源獎勵
4. 進入合成商店（或事件節點）
5. 拆解 / 合成 / 強化卡牌
6. 進入下一場戰鬥

> **重要原則：**
> - 戰鬥中不進行複雜組合
> - 深度決策集中在戰鬥外

---

## 3. 卡牌系統總覽

### 3.1 卡牌雙態設計

每張卡牌具備兩個層面：

- **使用面（戰鬥）**
  - 費用（Energy Cost）
  - 基礎效果（傷害 / 防禦 / 治療等）
  - 屬性與標籤

- **結構面（合成）**
  - 提供的合成零件（Components）
  - 可用於合成高階卡牌

---

### 3.2 卡牌資料結構（概念）

```
Card {
  Cost,
  BaseEffect,
  Tags[],        // 決定戰鬥互動與合成相容性
  Components[]  // 拆解後提供的零件
}
```

---

## 4. 第一層基礎卡牌（MVP）

### 4.1 設計原則
- 所有基礎卡 **可單獨使用**
- 每張卡至少提供 1 個合成零件
- 元素與型態嚴格受控，避免系統爆炸

### 4.2 使用的元素與型態

- **元素**：火 🔥 / 冰 ❄ / 物理 ⚔
- **型態**：近戰 🗡 / 遠程 🏹 / 防禦 🛡
- **功能**：強化 🔧 / 抽牌 🧠 / 治療 💚

---

### 4.3 基礎卡牌列表（12 張）

#### 元素系（4）
- 火球術（2 費｜8 火傷｜🔥 遠程）
- 火焰附魔（1 費｜攻擊 +4 火傷）
- 冰箭（2 費｜6 冰傷＋緩速）
- 冰霜護盾（2 費｜6 護甲並保留）

#### 物理系（4）
- 斬擊（1 費｜6 物理傷害）
- 突刺（1 費｜4 無視護甲）
- 射擊（1 費｜5 遠程傷害）
- 防禦（1 費｜5 護甲）

#### 功能系（4）
- 集中（1 費｜下一張牌 -1 費）
- 蓄力（1 費｜棄 1 牌，下一攻擊 +6）
- 治療（2 費｜回復 7 HP）
- 戰鬥專注（1 費｜抽 2 張）

---

## 5. 合成系統（第一層）

### 5.1 基本合成邏輯

- 任兩張卡，若 **Components / Tags 相容**，即可合成新卡
- 合成後卡牌：
  - 效果質變或使用方式改變
  - 成本通常提高

---

### 5.2 第一層合成路線

#### 元素 × 型態
- 火 + 近戰 → 火焰斬擊
- 火 + 遠程 → 火焰箭雨
- 冰 + 遠程 → 寒冰狙擊
- 冰 + 防禦 → 冰牆

#### 元素 × 元素
- 火 + 火 → 烈焰法術（高 DOT）
- 冰 + 冰 → 極寒法術（強控）
- 火 + 冰 → 蒸氣爆發（AOE 混亂）

#### 型態 / 功能 × 修飾
- 斬擊 + 蓄力 → 致命斬
- 射擊 + 集中 → 精準射擊
- 防禦 + 治療 → 回復護甲

---

## 6. 基礎數值平衡框架（Act 1 基準）

### 6.1 戰鬥價值定義

> **1 Energy ≈ 5 點戰鬥價值**

此價值可轉換為不同效果。

### 6.2 數值換算表

| 類型 | 1 費用 ≈ |
|----|----|
| 傷害 | 5 |
| 護甲 | 6 |
| 治療 | 4 |
| 抽牌 | 1.5–2 / 張 |
| 控制效果 | 消耗 2–4 傷害價值 |

> 屬性與未來潛力技能通常即時數值下修 10–20%。

---

## 7. 怪物數值反推基準（Act 1）

### 玩家假設
- 每回合 Energy：3
- 平均輸出：~15 傷害 / 回合

### 怪物設計基準

| 類型 | HP | 單回合傷害 |
|----|----|----|
| 普通怪 | 20–25 | 6–8 |
| 精英怪 | 45–60 | 10–14 |
| Boss | 150+ | 15+（含節奏機制） |

---

## 8. 設計約束（防止系統失控）

- 單張卡最多提供 2–3 個 Components
- 禁止修飾 × 修飾的無限疊加
- 治療卡必須稀少且偏弱
- 高階合成卡拆解成本提高

---

## 9. 本文件適用範圍

- 本文件定義 **MVP / Act 1** 的設計基準
- 後續角色差異、進階合成與高階元素，需在此框架上延伸

---

## 10. 後續可討論與延伸項目

- 角色職業與合成規則差異
- 第二層 / 第三層合成溢價公式
- 合成商店 UI / UX 流程
- 數值模擬與實際遊玩測試回饋

---

> **設計哲學總結：**
> 與其追求單卡平衡，不如確保「選擇本身有代價」。
